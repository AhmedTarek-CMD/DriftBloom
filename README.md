# Drift & Bloom

A fully responsive e-commerce frontend for **Drift & Bloom**, a plant / candle / gift
package brand. Built with React, Tailwind CSS, React Router DOM and React Icons.
No backend — all data is local, and Login / Register / Admin are UI-only mocks.

## Getting started

```bash
npm install
npm run dev
```

Then open the printed local URL (usually `http://localhost:5173`).

To build for production:

```bash
npm run build
npm run preview
```

## Demo accounts

- **Customer Login / Register** (`/login`, `/register`) — there's no real backend,
  so any name / email / password combination will "log you in".
- **Admin Login** (`/admin/login`) — use the demo credentials shown on the page:
  - Email: `admin@driftandbloom.com`
  - Password: `admin123`

## Pages

| Route | Description |
| --- | --- |
| `/` | Home — hero, USP strip, categories, best sellers, about, quiz promo |
| `/packages` | Packages — search, category filter, sort, responsive grid |
| `/packages/:id` | Product details — gallery, plant options, quantity, related products |
| `/cart` | Cart — line items, order summary, checkout form, order confirmation |
| `/find-your-soul` | A short multi-step quiz that recommends a package |
| `/support` | Contact form, contact info, FAQ accordion |
| `/login`, `/register` | Customer auth (mocked) |
| `/admin/login` | Admin auth (mocked, see credentials above) |
| `/admin` | Admin dashboard — stats + recent orders |
| `/admin/products` | Admin packages — search, add, edit, delete (session only) |
| `/admin/orders` | Admin orders — search + status filter |
| `/admin/customers` | Admin customers list |

## Notes

- **No animations or transitions** are used anywhere, per the brief — all state
  changes (menus, accordions, tabs) are instant.
- **Images** are on-brand placeholder graphics generated by placehold.co
  (beige/olive colored boxes with labels). Swap the URLs in `src/data/products.js`
  and elsewhere for real product photography whenever you're ready.
- **Currency** is shown in Egyptian Pounds (`LE`) to match the reference design.
- The cart and auth state persist to `localStorage` so refreshing the page doesn't
  lose your cart or session — there is otherwise no backend or real authentication.
